# อัพเดทเอกสาร (Update Documentation)

เมื่อผู้ใช้สั่งว่า **"อัพเดทเอกสาร"** ให้คุณ (AI) ทำตามขั้นตอนดังนี้อัตโนมัติ:

## 1. วิเคราะห์บริบทล่าสุด
- ตรวจสอบว่าในรอบการทำงานล่าสุด คุณเพิ่งเขียน/แก้ไขโค้ดเรื่องอะไรไป
- สรุปว่าการแก้ไขนั้นส่งผลต่อ **กฎของระบบ**, **โครงสร้างระบบ**, หรือ **UI/ฟีเจอร์ใหม่**

## 2. เลือกอัพเดทไฟล์ .md ที่เกี่ยวข้อง
คุณต้องใช้เครื่องมือ `view_file` เพื่ออ่าน และ `replace_file_content` เพื่อแก้ไขไฟล์เหล่านี้ (เลือกเฉพาะไฟล์ที่เข้าข่าย):

- 📄 **`skill.md`**: หากแก้ไขกฎการคำนวณเงินเดือน, หักสาย, OT, การสลับกะ, หรือเพิ่มฟีเจอร์ใหม่ ให้คุณไปเพิ่มบันทึกการแก้ไขลงในส่วน **`## Change Log`** ท้ายไฟล์เสมอ และปรับปรุงเนื้อหากฎด้านบนให้ตรงกับความจริง
- 📄 **`architecture.md`**: หากมีผลกับภาพรวมระบบ โครงสร้างไฟล์ หรือการเก็บข้อมูล (Data Persistence)
- 📄 **`migration_tasks.md`**: หากเกี่ยวข้องกับกระบวนการย้ายระบบ

## 3. แจ้งผลการอัพเดท
- เมื่อทำการแก้ไขไฟล์ .md เสร็จสิ้นแล้ว ให้สรุปให้ผู้ใช้ฟังว่าได้ทำการอัพเดทไฟล์ไหนไปบ้าง และเขียนข้อความอะไรลงไป
